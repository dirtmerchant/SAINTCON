*What is Vagrant?

  +Frontend to Hypervisor, but more
  +Development VMs as code

*What is Ansible?
 -http://www.ansible.com
  +The manual problem
  +Cofiguration as code
  +Playbooks
  +Inventory files

*Vagrant
 -vagrant_project directory
 -vagrant init
 -edit Vagrantfile
  +box name
  +forwarded port
  +shell script
 -vagrant up
 -vagrant status
 -bootstrap.sh
 -html directory
 -vagrant ssh default
 -vagrant destroy

*Ansible
 -Vagrantfile
  +Mgmt server
   =bootstrap.sh
  +Load balancer
  +Multiple web servers
 -vagrant up
 -vagrant status
 -vagrant ssh mgmt
 -ls -l /vagrant/examples
 -ansible --version
 -ssh web1
 -ansible web1 -m ping
 -ssh-keyscan web1
 -ssh-keyscan lb web1 web2 >> .ssh/known_hosts
 -cat inventory.ini
 -ansible all -m ping --ask-pass
 -ansible all -m ping
